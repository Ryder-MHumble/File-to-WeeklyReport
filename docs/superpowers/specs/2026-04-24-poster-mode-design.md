# 海报模式设计说明

日期：2026-04-24

## 1. 目标

在不影响既有周报主链路稳定性的前提下，为系统新增一个独立的“海报模式”，支持用户基于文件或文本输入，生成研究院对内 / 对外宣传海报。

本次设计强调三个约束：

- 周报模式与海报模式在 UI 和控制器层面隔离，避免互相污染状态
- 海报模式优先复用既有 OpenRouter 代理能力，不新增高风险服务端新路由
- 当模型不可用时，仍可回退到本地草图，保证联调和演示可用

## 2. 方案选择

### 方案 A：复用现有页面，增加业务模式切换

- 做法：在顶层增加 `productMode`，分别挂载 `ReportWorkbench` 与 `PosterWorkbench`
- 优点：变更集中、隔离清晰、回归面可控
- 缺点：需要拆分原有 `App.jsx` 结构

### 方案 B：在原周报工作台内继续叠加海报控件

- 做法：直接在现有周报控制台里扩字段、扩状态、扩预览区
- 优点：实现快
- 缺点：状态耦合重，回归风险高，后续维护成本大

### 方案 C：新增单独路由或新页面

- 做法：引入路由系统或新增独立页面入口
- 优点：边界更硬
- 缺点：当前项目没有路由基础设施，改动面偏大

推荐采用方案 A。

## 3. 架构设计

### 3.1 顶层结构

- `App.jsx`
  - 维护 `productMode`
  - 根据模式切换到 `ReportWorkbench` 或 `PosterWorkbench`
- `AppHeader.jsx`
  - 提供模式切换入口
  - 保留统一品牌头部

### 3.2 海报模式模块

- `PosterWorkbench.jsx`
  - 海报工作台容器
  - 负责控制区、来源区、生成区、预览区装配
- `PosterPreferenceSection.jsx`
  - 海报场景、风格、部门、比例、尺寸、敏感模式配置
- `PosterPreviewPane.jsx`
  - 空态、生成中、生成完成三态展示
  - 支持下载与最近生成记录
- `use-poster-workbench-controller.js`
  - 海报模式专属状态、事件、日志、回退和下载控制

### 3.3 海报生成链路

1. 输入文本或多文档内容
2. 复用 `file-extractor` 做文本抽取与合并
3. 通过 `OPENROUTER_POSTER_BRIEF_MODEL` 输出海报 brief JSON
4. 基于 brief、场景、风格、比例、尺寸拼装图片 prompt
5. 通过 `OPENROUTER_POSTER_IMAGE_MODEL` 生成海报图片
6. 前端展示图片、摘要、关键信息，并允许下载

## 4. 提示词与风格包

### 4.1 brief JSON 结构

- `title`
- `subtitle`
- `headline`
- `supporting_copy`
- `key_points`
- `visual_subject`
- `layout_focus`
- `cta`
- `negative_prompts`

### 4.2 风格包来源

- 海报场景：约束内容表达目标和受众
- 海报风格：约束视觉气质、色彩方向和 prompt hint
- 部门信息：补充机构语义
- 敏感表达模式：控制措辞强度和宣传边界

## 5. 稳定性与回退

### 5.1 隔离策略

- 周报与海报分别使用独立 workbench 和 hook
- 生成按钮文案复用现有组件，但通过参数控制行为
- 不修改现有周报生成主链路逻辑

### 5.2 回退策略

- 未配置 API Key：直接回退本地 SVG 草图
- 图片模型失败：保留 brief 并回退本地草图
- brief 内容解析不足：基于原文首屏信息构造 fallback brief

## 6. 验证策略

### 6.1 周报回归

- 切回周报模式
- 输入真实文本
- 触发生成
- 验证模板预览、导出按钮、复制链接按钮恢复正常

### 6.2 海报验证

- 在生产服务代理打开情况下进行真实模型调用
- 验证 brief 生成、海报图片展示、下载动作
- 验证 usage 记录中写入 brief 模型和图片模型两条调用

## 7. 验收标准

- 可在同一入口切换“周报模式 / 海报模式”
- 海报模式支持文本与多文档输入
- 海报模式支持场景、风格、比例、尺寸、敏感模式配置
- 海报模式可通过 OpenRouter 成功生成图片
- 海报模式失败时可回退到本地 SVG 草图
- 周报模式主链路无回归
